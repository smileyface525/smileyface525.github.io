Functional vs Object Oriented Programming

April 20, 2014

In general, programming is all about taking actions with information(data). Good example would be a HTML form: user enters her information into a form, a program validates it, then stores the information in the database. You can ultimately say that what the program is doing is just taking nouns(data) and using verbs to do something with the nouns. In functional programming, its focus is on the verbs. Do this, then do that. With OOP, the focus is on the nouns, and that is made possible by using classes and objects: Class being a bluprint of what an object is supposed to do.

Easiest way to see the differences is to look at some expamples...

First, let's assume that we have a online form which collects a few imformation from users: Fist name, last name and also username and password of their choice. And we want to organize the information in some way so that ultimately we can connect to the database and permanetly store that information.

In OOP...

Class Users

attr_accessor :first_name, :last_name, :username, :password

def initialize(first_name, last_name)
@first_name = first_name
@last_name = last_name
@username = '' @password = ''
end

def password
@password
end

end

eiko = Users.new('eiko', 'seino') # new instance of Users is created
eiko.username=('eiko's username') # username called 'eiko's username' is created
eiko.password=('eiko's password') # password called 'eiko's password' is created
eiko.lastname # => 'seino'
eiko.username # => 'eiko's username'

In functional programming...

first_name = 'eiko'
last_name = 'seino'
username = 'eiko's username'
password = 'eiko's password'
puts first_name # => 'eiko'
puts last_name # => 'seino'

The real difference shows if we have multiple users. Say we have another user who has filled out the form.
In OOP...

# Let's say Alex Parish has filled out the form and submitted it

alex = Users.new('alex', 'parish')
alex.username=('alex's username') # username called 'alex's username' is created
alex.password=('alex's password') # password called 'alex's password' is created
alex.lastname # => 'parish'
alex.username # => 'alex's username'

In functional programming...

alex_first_name = 'alex'
alex_last_name = 'parish'
alex_username = 'alex's username'
alex_password = 'alex's password'
puts alex_first_name # => 'alex'
puts alex_last_name # => 'parish'

The difference may not be so obvious but there are a few crucial differences between the two.
When we tried to add Eiko, the OOP version of the code is longer since we had to define the class itself. However, when we added Alex, notice that we only had to create an instance of alex using the same class Users, and we were able to change and access his information just by an calling appropriate method.
On the other hand, iin functional programming, not only we had do excalty the same thing we needed to do with Eiko but also we had to change the variable names so that we don't override Eiko's information.

So, what does this mean? Is OOP better than functional programming?

Not necessarily. It just means that if we're in a situation that we have to so the same things to a different nouns(objects), OOP may be the better choice. But if we need to work on one particular thing and do many different things with it(by usings lots of verbs), functional programming maight be the way to go.
